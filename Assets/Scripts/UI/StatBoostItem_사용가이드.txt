===========================================
영구 스탯 증가 아이템 시스템 사용 가이드
===========================================

이 시스템은 플레이어의 스탯을 영구적으로 증가시키는 아이템을 만들 수 있습니다.

===========================================
1. ScriptableObject 생성 (아이템 데이터)
===========================================

Unity Editor에서:
1. Project 창에서 우클릭
2. Create → Inventory → Stat Boost Item 선택
3. 파일 이름 지정 (예: "AttackPowerBoost", "DefenseBoost" 등)

Inspector에서 설정:
- Item Name: 아이템 이름 (예: "공격력 물약")
- Description: 아이템 설명 (예: "공격력을 5 증가시킵니다")
- Icon: 아이템 아이콘 스프라이트
- Boost Type: 스탯 타입 선택
  • AttackPower: 공격력만 증가
  • Defense: 방어력만 증가
  • MaxHealth: 최대 체력만 증가
  • MoveSpeed: 이동 속도만 증가
  • AllStats: 모든 스탯 조금씩 증가

- 증가량 설정:
  • Attack Power Boost: 공격력 증가량 (예: 5)
  • Defense Boost: 방어력 증가량 (예: 2)
  • Max Health Boost: 최대 체력 증가량 (예: 20)
  • Move Speed Boost: 이동 속도 증가량 (예: 0.5)

- Use Sound: 사용 시 재생할 사운드
- Use Effect Prefab: 사용 시 생성할 이펙트 (선택사항)

===========================================
2. 프리팹 생성 (월드에 배치할 아이템)
===========================================

방법 1: 즉시 사용되는 아이템 (권장)
--------------------------------------
1. Hierarchy에서 빈 GameObject 생성
2. 이름 지정 (예: "AttackBoostPickup")
3. 컴포넌트 추가:
   - SpriteRenderer: 아이템 비주얼
   - Rigidbody2D:
     * Body Type: Dynamic
     * Gravity Scale: 1
     * Constraints: Freeze Rotation Z 체크
   - CircleCollider2D 또는 BoxCollider2D:
     * Is Trigger: 체크
   - StatBoostItemPickup 스크립트

4. StatBoostItemPickup 설정:
   - Stat Boost Item: 위에서 만든 ScriptableObject 할당
   - Add To Inventory: false (즉시 사용)
   - Sprite Renderer: SpriteRenderer 컴포넌트 할당
   - Ground Layer: Ground 레이어 선택
   - Pickup Effect Prefab: 획득 시 이펙트 (선택사항)
   - Pickup Sound: 획득 시 사운드 (선택사항)

5. Prefab으로 저장

방법 2: 인벤토리에 추가되는 아이템
--------------------------------------
위와 동일하지만:
- Add To Inventory: true 로 설정
- 플레이어가 인벤토리에서 직접 사용해야 함

===========================================
3. 몬스터 드롭 설정
===========================================

몬스터가 죽을 때 스탯 아이템을 드롭하도록 설정:

1. 몬스터 GameObject 선택
2. ItemDrop 컴포넌트 찾기
3. Item Drop List에 새 항목 추가:
   - Item Prefab: 위에서 만든 프리팹 할당
   - Drop Chance: 드롭 확률 (0~100%) - 예: 5% = 희귀
   - Min Drop Count: 최소 드롭 개수 (보통 1)
   - Max Drop Count: 최대 드롭 개수 (보통 1)

권장 드롭 확률:
- 일반 몬스터: 1~5% (매우 희귀)
- 중간 보스: 10~20% (희귀)
- 보스: 50~100% (확정 또는 높은 확률)

===========================================
4. 스탯 타입별 추천 설정
===========================================

공격력 물약 (AttackPower):
- attackPowerBoost: 3~5
- 설명: "공격력을 영구적으로 증가시킵니다"

방어력 물약 (Defense):
- defenseBoost: 2~3
- 설명: "방어력을 영구적으로 증가시킵니다"

체력 물약 (MaxHealth):
- maxHealthBoost: 20~30
- 설명: "최대 체력을 영구적으로 증가시킵니다"

속도 물약 (MoveSpeed):
- moveSpeedBoost: 0.3~0.5
- 설명: "이동 속도를 영구적으로 증가시킵니다"

만능 엘릭서 (AllStats):
- attackPowerBoost: 2
- defenseBoost: 1
- maxHealthBoost: 10
- moveSpeedBoost: 0.2
- 설명: "모든 능력치를 향상시키는 귀한 물약"

===========================================
5. 테스트 방법
===========================================

1. Scene에 스탯 아이템 프리팹 배치
2. Play 모드 실행
3. 플레이어로 아이템에 접근
4. Console에서 스탯 증가 메시지 확인
5. UI에서 스탯 변화 확인

또는:

1. 몬스터를 처치
2. 아이템 드롭 확인 (낮은 확률이므로 여러 번 처치 필요)
3. 드롭된 아이템 획득
4. 스탯 증가 확인

===========================================
6. 주의사항
===========================================

- 스탯 증가는 영구적이며 되돌릴 수 없습니다
- 게임 밸런스를 위해 증가량을 적절히 조절하세요
- 드롭 확률을 너무 높게 설정하면 게임이 쉬워질 수 있습니다
- PlayerController, PlayerStats, PlayerHealth가 필요합니다
- "Player" 태그가 플레이어에게 설정되어 있어야 합니다

===========================================
7. 코드 통합 정보
===========================================

생성된 파일:
- StatBoostItemData.cs: 아이템 데이터 정의
- StatBoostItemPickup.cs: 월드 픽업 처리

기존 시스템 활용:
- ItemDrop.cs: 몬스터 드롭 (수정 불필요)
- PlayerStats.cs: bonusAttackPower, bonusMoveSpeed 사용
- PlayerHealth.cs: defense, AddPermanentHealth() 사용
- Inventory.cs: 인벤토리 시스템 (선택적 사용)

===========================================
8. 예제 시나리오
===========================================

시나리오 1: 숨겨진 보상
- 맵에 숨겨진 공격력 물약 배치
- 플레이어가 탐험하면서 발견
- 즉시 사용되어 공격력 +5

시나리오 2: 보스 보상
- 보스 몬스터가 확정으로 만능 엘릭서 드롭
- 플레이어 모든 스탯 소폭 증가
- 보스 처치의 보람

시나리오 3: 희귀 드롭
- 일반 몬스터가 1% 확률로 체력 물약 드롭
- 운이 좋으면 최대 체력 +20
- 파밍 요소 추가
